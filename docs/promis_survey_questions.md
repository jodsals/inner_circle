# PROMIS-Basierter Fragebogen für Inner Circle App

## Übersicht

Dieses Dokument definiert die wissenschaftlichen Fragen basierend auf PROMIS (Patient-Reported Outcomes Measurement Information System) Domänen für die Inner Circle App.

## Zeitpunkte

- **Baseline (T0)**: Bei Registrierung
- **Follow-up (T1)**: Nach 8 Wochen

## Antwortkategorien

### Häufigkeitsskala (Frequency Scale)

1. Nie
2. Selten
3. Manchmal
4. Oft
5. Immer

### Intensitätsskala (Intensity Scale)

1. Überhaupt nicht
2. Ein wenig
3. Mäßig
4. Ziemlich
5. Sehr

### Zustimmungsskala (Agreement Scale)

1. Stimme überhaupt nicht zu
2. Stimme eher nicht zu
3. Neutral
4. Stimme eher zu
5. Stimme völlig zu

## Follow-up Spezifische Fragen (Nur bei T1 nach 8 Wochen)

### APP-1

**Frage**: Hat Ihnen die Inner Circle App in den letzten 8 Wochen geholfen?
**Skala**:

1. Überhaupt nicht geholfen
2. Kaum geholfen
3. Teilweise geholfen
4. Ziemlich geholfen
5. Sehr geholfen

### APP-2

**Frage**: In welchen Bereichen hat Ihnen die App am meisten geholfen? (Mehrfachauswahl möglich)
**Optionen**:

- [ ]  Gefühl der Zugehörigkeit zu einer Gemeinschaft
- [ ]  Emotionale Unterstützung durch andere Mitglieder
- [ ]  Zugang zu wertvollen Informationen und Ressourcen
- [ ]  Reduzierung von Einsamkeitsgefühlen
- [ ]  Aufbau neuer sozialer Kontakte
- [ ]  Verbesserung meiner psychischen Gesundheit
- [ ]  Besseres Verständnis meiner Situation
- [ ]  Motivation für positive Veränderungen
- [ ]  Sonstiges (bitte angeben)

### APP-3

**Frage**: Wie häufig haben Sie die App in den letzten 8 Wochen genutzt?
**Skala**:

1. Nie oder fast nie
2. Ein paar Mal im Monat
3. Ein paar Mal pro Woche
4. Fast täglich
5. Täglich oder mehrmals täglich

### APP-4

**Frage**: Würden Sie die Inner Circle App anderen Personen in einer ähnlichen Situation empfehlen?
**Skala**:

1. Definitiv nicht
2. Wahrscheinlich nicht
3. Vielleicht
4. Wahrscheinlich ja
5. Definitiv ja

### APP-5 (Optional - Offene Frage)

**Frage**: Was hat Ihnen an der Inner Circle App am besten gefallen und was könnte verbessert werden?
**Format**: Freitext

## Scoring und Auswertung

### Berechnung der Domain-Scores

Für jede PROMIS-Domäne:

1. Summiere die Rohwerte aller Items der Domäne
2. Berechne den Durchschnittswert: Summe / Anzahl der Items
3. Optional: Konvertiere zu PROMIS T-Score (M=50, SD=10)

### Interpretation der Veränderungen (T0 zu T1)

**Klinisch bedeutsame Verbesserung**:

- Reduktion um ≥5 T-Score Punkte bei Depression, Angst, Sozialer Isolation
- Erhöhung um ≥5 T-Score Punkte bei Emotionaler Unterstützung, Sozialer Teilhabe

**Minimale klinisch bedeutsame Differenz (MCID)**:

- Depression: 5-8 Punkte
- Angst: 4-6 Punkte
- Soziale Isolation: 5-7 Punkte
- Emotionale Unterstützung: 4-6 Punkte

## Implementierungsempfehlungen

### Baseline-Fragebogen (T0 - Registrierung)

**Gesamtzahl**: 27 Fragen
**Geschätzte Dauer**: 8-12 Minuten
**Domänen**:

- Depression (5 Fragen)
- Angst (5 Fragen)
- Soziale Isolation (5 Fragen)
- Emotionale Unterstützung (5 Fragen)
- Soziale Teilhabe (4 Fragen)
- Lebensqualität (3 Fragen)

### Follow-up Fragebogen (T1 - Nach 8 Wochen)

**Gesamtzahl**: 32 Fragen
**Geschätzte Dauer**: 10-15 Minuten
**Domänen**:

- Alle Baseline-Fragen (27 Fragen)
- App-Bewertung (5 Fragen)

### Technische Umsetzung

1. **Datenmodell**:

```dart
class SurveyResponse {
  String userId;
  String surveyId;
  DateTime completedAt;
  SurveyType type; // baseline | followup
  Map<String, int> domainScores;
  List<QuestionResponse> responses;
  AppEffectivenessResponse? appFeedback; // nur bei followup
}

class QuestionResponse {
  String questionId;
  String domain;
  int score;
}

class AppEffectivenessResponse {
  int overallHelpfulness; // APP-1
  List<String> helpfulAreas; // APP-2
  int usageFrequency; // APP-3
  int recommendationLikelihood; // APP-4
  String? openFeedback; // APP-5
}
```

2. **Erinnerungssystem**:

   - Speichere Registrierungsdatum
   - Berechne 8-Wochen-Termin
   - Sende Push-Benachrichtigung/E-Mail 7 Tage vor Ablauf
   - Erinnere erneut am Zieldatum
   - Letzte Erinnerung 7 Tage nach Zieldatum
3. **Datenschutz**:

   - Pseudonymisierung der Antworten
   - Verschlüsselte Speicherung
   - DSGVO-konform
   - Opt-out Möglichkeit
4. **UI/UX Empfehlungen**:

   - Fortschrittsanzeige
   - Möglichkeit zu pausieren und später fortzusetzen
   - Überspringen-Option für sensible Fragen
   - Klare Erklärung des Zwecks
   - Ergebnisse in verständlicher Form anzeigen (optional)

## Ethische Überlegungen

1. **Informed Consent**: Nutzer müssen über Zweck und Verwendung der Daten aufgeklärt werden
2. **Freiwilligkeit**: Teilnahme muss freiwillig sein
3. **Datenschutz**: Höchste Sicherheitsstandards
4. **Krisenintervention**: Bei kritischen Antworten (z.B. Suizidgedanken) → Notfall-Ressourcen anzeigen
5. **Transparenz**: Klare Kommunikation über Studienzweck und Datenverwendung

---

## Validität und Reliabilität

Alle verwendeten Fragen basieren auf validierten PROMIS Item Banks:

- **Validität**: Inhaltlich, konstruktiv und kriteriumsbezogen validiert
- **Reliabilität**: Cronbach's Alpha > 0.90 für alle Domänen
- **Sensitivität**: Nachgewiesene Fähigkeit, Veränderungen über Zeit zu detektieren
- **Normwerte**: Verfügbar für Vergleich mit Allgemeinbevölkerung
